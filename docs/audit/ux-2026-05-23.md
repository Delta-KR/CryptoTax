# UI/UX + a11y audit — 2026-05-23

Scope: marketing landing, auth, and dashboard app (Next.js 14 / Tailwind / Korean).
Baseline: DESIGN.md tokens, WCAG AA, 375px mobile minimum.

## P0 (must fix — breaks usability or a11y)

### Keyboard accessibility

- **Transactions rows expand only via click, not keyboard** — `app/(app)/transactions/page.tsx:232-235`
  Each row uses `<TableRow onClick={() => toggleExpand(tx.id)} className="cursor-pointer">` but has no `tabIndex`, no `role="button"`, no `onKeyDown`. Keyboard users cannot see the rate audit detail panel. The chevron icon hints at expansion but is decorative.
  Fix: wrap the chevron+date cell content in a real `<button>` (or add `role="button"`, `tabIndex={0}`, `onKeyDown` that handles Enter/Space) and `aria-expanded={isOpen}` + `aria-controls`.

- **App layout has no skip link** — `components/app-chrome/AppShell.tsx:18-35`
  The marketing `app/layout.tsx:156` injects `<a href="#main" class="skip-link">` and marketing/auth/app `<main id="main">` elements all exist, so the skip link works on marketing/auth — but for app shell it lands inside a 240px sidebar (8+ links) before any content. Skip link works only because main has id `main`. Verify it actually jumps past sidebar (it does, by virtue of being at body root). Action: no fix required if skip link is verified to bypass sidebar, but ensure the skip link is visible enough on dark theme (currently uses solid brand bg + white text — OK).
  → Verified clean.

- **Sidebar / Topbar / Modal / Toast z-index collisions** — `components/ui/Modal.tsx:45` / `components/ui/Toast.tsx:64` / `components/app-chrome/Topbar.tsx:13` / `components/app-chrome/MobileDrawer.tsx:29`
  Modal = `z-50`, Toast = `z-50`, MobileDrawer = `z-50`, Topbar = `z-40`. When a Modal opens, a Toast triggered while open paints at the same z-index — render order decides which wins. With ProfilePage delete-confirm Modal + error Toast, the toast can render behind the modal backdrop.
  Fix: bump Toast to `z-[70]` (above modal), keep Modal `z-[60]` to match DESIGN.md.

- **Dashboard quick-action card uses purple (#7C3AED) — DESIGN.md violation + brand consistency** — `app/(app)/dashboard/page.tsx:74` and `app/(app)/settings/page.tsx:30`
  DESIGN.md §3 explicitly forbids 보라(#7C3AED) outside exchange brand colors. Two dashboards use it as a "세금 계산" icon accent and Settings uses it for "알림 설정". Theme tokens should drive this.
  Fix: replace with `text-brand` + `bg-brand-soft` (or use a `text-step-2` token if a distinct hue is needed — but the design system clearly says single brand blue only).

- **Tax / dashboard premium upsells use brand→purple gradient** — `app/(app)/tax/page.tsx:92, 760, 829, 892` and `app/(app)/dashboard/page.tsx:229` and `app/(app)/billing/page.tsx:43`
  `linear-gradient(135deg, rgb(var(--brand)) 0%, rgb(124,58,237) 100%)` (purple) and `linear-gradient(165deg, #1E3A8A 0%, #0F1B3D 100%)` are direct DESIGN.md §8 "AI 티" anti-patterns ("그라디언트 텍스트 — 브랜드→인디고→보라", "다크 그라디언트 카드 (#1E3A8A → #0F1B3D)").
  Fix: single brand surface (`bg-brand`) + brand-glow shadow already defined in `tailwind.config.ts:81`. Drop the violet stop.

### Forms & validation

- **Login error renders on password field only — confusing when error is about email** — `app/(auth)/login/page.tsx:111`
  `<Input ... error={error ?? undefined} />` attaches the form-level error string to the password field. If error is "이메일과 비밀번호를 모두 입력해주세요" but user only forgot to type email, the error appears under password.
  Fix: render the form-level error in a dedicated error banner above the form (mirror the `oauthError` banner at line 85), keep field-level errors for field-specific validation.

- **Signup field-level errors collide on the password field** — `app/(auth)/signup/page.tsx:192`
  Same pattern — name/email/agree errors all funnel into the password Input. A user who didn't agree to terms sees "이용약관에 동의해주세요" under the password.
  Fix: split into form-level banner + per-field error state (`nameError`, `emailError`, `agreeError`).

## P1 (significant degradation)

### Accessibility

- **`muted-2` (#8A93A4) text on light backgrounds fails WCAG AA in many places** — `app/globals.css:45`
  Light mode: #8A93A4 on white (#FFF) ≈ 3.2:1, on `bg-soft` (#FAFBFD) ≈ 3.1:1. WCAG AA needs 4.5:1 for normal text. Used as body subtext (sub captions in StatCard `components/ui/StatCard.tsx:57`, CTA stats `components/sections/cta.tsx:40`, footer `components/sections/footer.tsx:90`, table headers `components/ui/Table.tsx:44`, pricing footer `components/sections/pricing.tsx:185`, Roadmap disclaimer `components/sections/roadmap.tsx:127`, exchange request line `components/sections/exchanges.tsx:149` — 80+ call sites).
  Fix: redefine `--muted-2` to ~`107 114 128` (#6B7280, ratio 4.6:1) or restrict its use to non-essential 11px eyebrow labels that are part of a labeled group.

- **Hero macOS chrome dots are decorative red/yellow/green but have no semantic ARIA** — `components/sections/hero.tsx:198-201`
  Three colored dots imply window controls but aren't interactive. Light readers will read empty spans. Acceptable as decoration but the parent flex doesn't have `aria-hidden="true"`. Add to the wrapping div.

- **Sidebar active-link indicator relies only on bg/text color** — `components/app-chrome/Sidebar.tsx:115-117`
  `aria-current="page"` is set (good), but for color-blind users the only additional signal is `bg-brand-faint` + `text-brand`. The 3px left bar at line 121 reinforces it. OK in light mode — verify in dark mode where `--brand-faint` (#182238) is very close to `--card` (#131C2E) — the active state may be nearly invisible. Recommend bumping the 3px bar contrast in dark.

- **Theme toggle flashes wrong icon for first 1–2 frames before mount** — `components/theme-toggle.tsx:32`
  Returns `mounted && dark ? <Sun /> : <Moon />`. While `mounted=false`, always shows moon. If user starts on dark mode → moon (wrong) → sun (correct) flash. Visible because the inline `bootScript` in `app/layout.tsx:101` sets `data-theme` before paint, but React state lags by one effect tick.
  Fix: SSR-time read of cookie/header, or render a placeholder div until mounted (same `h-9 w-9` to prevent layout shift, with `aria-label="테마 전환"` and no icon).

- **Modal close button is the whole backdrop, but it has `aria-label="닫기"` and intersects with content** — `components/ui/Modal.tsx:47-52`
  `<button aria-label="닫기" className="absolute inset-0 bg-black/40">` is below the panel — clicks pass through. OK functionally, but a screen reader treats the entire backdrop as a button. Better: backdrop = `<div aria-hidden>`, add an actual `×` close button in the title row (which is missing today).

- **Mobile drawer backdrop is an `<button>` but lacks focus trap** — `components/app-chrome/MobileDrawer.tsx:31-36`
  No focus management when drawer opens — keyboard focus stays where it was, can leak to Topbar behind the backdrop via Tab. The `mobile-nav.tsx` open-state has the same issue (no focus trap inside the dialog).
  Fix: on open, focus the first link in the panel; trap Tab cycle within the dialog (or use `inert` on background).

### Responsive / mobile

- **Hero floating cards are hidden < `md` (768px) — abrupt visual emptiness** — `components/sections/hero.tsx:134, 168` (`md:flex` / `md:block` + `hidden`)
  On mobile, the dashboard mock loses its two floating cards (upbit + PDF) entirely, leaving just the macOS-chrome screenshot. The "upbit 거래내역 통합 완료" + "tax_report_홍길동.pdf 세무사 전달 가능" trust signals are lost on the most-trafficked viewport.
  Fix: render scaled-down inline (below the dashboard mock) on mobile, or as a single combined chip.

- **Transactions table overflows horizontally on mobile — no `overflow-x-auto` wrapper** — `app/(app)/transactions/page.tsx:213-310`
  Eight columns at 13px is ~640px minimum. The `Table` component (`components/ui/Table.tsx:6`) wraps in `<div class="overflow-hidden">` which clips, not scrolls. On 375px mobile, columns get crushed.
  Fix: change the outer wrapper to `overflow-x-auto` (or build a card-list variant for mobile).

- **`Pricing` 3-col grid stacks `1fr_1.18fr_1fr` only at `lg:` — tablets (768-1023) get a single column** — `components/sections/pricing.tsx:173`
  No `md:` rule, so on iPad portrait the user sees 3 tall stacked pricing cards. Lots of scroll for a comparison surface.
  Fix: `md:grid-cols-3 lg:grid-cols-[1fr_1.18fr_1fr]` so tablets show comparison view too.

- **Roadmap 4-col stacks the same way — tablet sees 4 long stacked cards** — `components/sections/roadmap.tsx:121`
  Add `md:grid-cols-2 lg:grid-cols-4`.

- **Exchanges "COMING SOON" grid uses `w-[calc(50%-6px)] md:w-[calc(25%-9px)]`** — `components/sections/exchanges.tsx:143`
  At sub-md widths the 7 cards form 2 columns with one orphan. Functional but visually unbalanced; consider `flex-1 min-w-[140px]` so they auto-size.

- **Marketing Nav nav-links hidden at < md — but only "메뉴 열기" hamburger exposes them on mobile.** — `components/sections/nav.tsx:39, 76`
  The hamburger jumps to `MobileNav` sheet — OK. Confirm in `MobileNav` the layout above shows kakao/네이버/구글 login alternatives not exposed in MobileNav (currently only login/signup buttons are exposed).
  → Verified: SocialButtons live on /login + /signup, not on the nav drawer. Reasonable, but worth flagging.

### Theme / dark mode

- **`Hero` floating card `bg-card` + light `--logo-bg` hex (#EEF3FF) in dark mode** — `components/sections/hero.tsx:137`
  Inline style `background: '#EEF3FF'` is overridden by `[data-theme="dark"] .exchange-logo-bg` mixing rule — but this card uses an inline `style={{ background: '#EEF3FF' }}` directly, not the `.exchange-logo-bg` class. So in dark mode the upbit logo tile inside the floating card stays light-blue, breaking the dark palette.
  Fix: switch to `<div class="exchange-logo-bg" style={{ '--logo-bg': '#EEF3FF' }}>` to inherit the dark-mode color-mix.

- **macOS dots (#FF5F56, #FFBD2E, #27C93F) in `hero.tsx:199-201` are pure hex.**
  DESIGN.md §3 forbids raw hex. These are skeuomorphic window controls — acceptable exception but worth a comment.

- **`Pricing` middle card uses `border-brand` ring + brand-faint surface — verify in dark mode.** — `components/sections/pricing.tsx:102`
  Dark `--brand-faint` = #182238, very close to `--card` (#131C2E). The "emphasis" treatment may look identical to the surrounding cards. Combined with `BEST VALUE` brand badge on top it still reads, but the ring is essential — verify `ring-brand/10` is visible in dark.

- **Billing page premium card has hardcoded dark gradient `linear-gradient(165deg, #1E3A8A 0%, #0F1B3D 100%)`** — `app/(app)/billing/page.tsx:43`
  This is the literal DESIGN.md §8 anti-pattern. The card in dark mode is _below_ the page bg in lightness, looks like a hole. Replace with `bg-brand` + brand-glow shadow.

### Loading / error states

- **No `loading.tsx` files anywhere** — `app/`
  Next.js `loading.tsx` files at any route segment provide automatic Suspense boundaries. None exist, so route transitions show a blank page until the new content loads. Especially noticeable on `/billing`, `/transactions`, `/tax` where `useCurrentUser()` returns `null` on first paint (see `AppShell.tsx:16` — `if (loading || !user) return null;`).
  Fix: add `app/(app)/loading.tsx` rendering a minimal sidebar+topbar skeleton.

- **No `error.tsx` or `global-error.tsx` files** — `app/`
  An unhandled server-action throw or client-side render error currently shows the Next.js default error screen.
  Fix: add `app/error.tsx` + `app/(app)/error.tsx` with localized Korean fallback UI.

- **No `not-found.tsx`** — `app/`
  Visiting `/foo` shows the default Next.js 404. Add `app/not-found.tsx`.

- **`Skeleton` component exists but is unused** — `components/ui/Skeleton.tsx`
  Imported nowhere. Plenty of `if (loading) return null;` patterns (`AppShell.tsx:16`, `profile/page.tsx:173`). Replacing with Skeleton-based layouts would feel substantially snappier.

- **Transactions / dashboard / tax pages render with empty data until `useEffect` populates** — `app/(app)/transactions/page.tsx:106-108`
  `useState<Transaction[]>([])` + `useEffect → setAll(getTransactions())` causes a flash of "아직 거래 내역이 없습니다" empty state for users who do have transactions. The EmptyState then disappears once data loads. Misleading.
  Fix: gate the EmptyState behind a `loaded` flag (`useState(false)` → `true` in the effect), or read transactions synchronously before first paint if storage is `localStorage` (no SSR concern since these pages are client-only).

### Visual hygiene

- **Inconsistent border-radius** — `rounded-sm` (8px), `rounded-md` (12px), `rounded-lg` (16px), `rounded-[10px]`, `rounded-[14px]`, `rounded-[18px]`, `rounded-2xl` (28px) all appear within sections.
  DESIGN.md §6 standardizes `rounded-lg` (16px) for cards and `rounded-[18px]` for big cards. Today's reality:
  - Hero CTA button: `rounded-[10px]` (`hero.tsx:275`)
  - Nav login button: `rounded-sm` (8px) (`nav.tsx:69`)
  - Modal: `rounded-lg` (16px) (`Modal.tsx:55`)
  - AuthCard: `rounded-lg` (16px) (`AuthCard.tsx:15`)
  - Stat card: `rounded-lg` (16px) but with `rounded-[10px]` inner tiles
  - Pricing card: `rounded-[18px]` (`pricing.tsx:100`)
  - Mobile sheet panel: not explicit, inherits
  Fix: settle on `rounded-md` (12px) for buttons, `rounded-lg` (16px) for cards, `rounded-[18px]` for pricing/emphasized cards. Audit + sweep.

- **Inconsistent button padding** — `Button.tsx` `md` = `px-4 py-2.5` (text 14px), but Hero CTA uses `px-6 py-4` (text 15px), Nav login uses `px-4 py-[9px]`, CTA final uses `px-7 py-[15px]`.
  Five different button heights for the same "primary" intent. Centralize to two heights: nav (32px) and CTA (48px).

- **`hover:scale-105` / `hover:scale-110` on premium upsell buttons** — `app/(app)/tax/page.tsx:683, 757, 826, 889` and dashboard:226 and report:181
  DESIGN.md §7 forbids scale animations as "AI tier". Drop the scale, keep `hover:bg-brand-2` only.

- **`animate-pulse rounded-md blur-md` glow halos** — `app/(app)/tax/page.tsx:685, 764, 833, 896` and dashboard:233 and report
  Pulsing glow behind premium CTA buttons is exactly the "iOS notification" anti-pattern. Replace with a subtle `bg-brand/10` static halo or remove.

- **PremiumBanner uses `shadow-[0_10px_30px_-12px_rgba(37,99,235,0.5)]` heavy brand glow** — `app/(app)/tax/page.tsx:89`
  Heavy shadow + 3-stop gradient = AI-tier. Use `shadow-brand-glow` token (already defined in `tailwind.config.ts:81`).
