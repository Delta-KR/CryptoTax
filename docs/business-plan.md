## 1. 시장과 문제

### 시점 — 2027.01.01 시행, 2028.05 첫 확정신고

2027년 1월 1일부터 가상자산 양도·대여 소득은 **기타소득으로 분리과세** (국세청 공식 안내).

| 항목 | 값 |
|------|---|
| 과세 표준 | 양도차익 250만원 초과분 |
| 세율 | 22% (지방세 포함) |
| 신고 시점 | 양도분 다음 해 5월 확정신고 → **첫 확정신고 2028.05** |
| 근거 법령 | 소득세법 §94, 시행령 §88, 국세기본법 §47의2~4 (가산세) |

### 시장 규모 (2026 추정)

| 단위 | 추정 | 출처 |
|------|------|------|
| 거래가능 이용자 (TAM) | **1,150만 ~ 1,250만명** | 금융위·금감원 실태조사 연율화 (2024.6 778만 → 2024말 970만 → 2025.6 1,077만) |
| 신고 의무 발생자 — 양도차익 250만 초과 (SAM) | **90만 ~ 230만명** (보유자 8 ~ 20%) | 100만원 미만 계좌 804만개 (2025.6 전체의 75%+) 분포 반영 보수 추정 |
| 1인당 평균 보유액 | **약 658만원** (2024.11 5대 거래소) | 한국은행 자료 인용 (뉴스1 보도) |
| 1인당 평균 양도차익 (전체 보유자) | **80 ~ 400만원** | 신고대상 비율 × 신고대상 평균 차익 1,000 ~ 2,000만 가정 |

**자료 한계**: 공식 "양도차익 250만 초과 비율" 통계는 부재. 위 수치는 보유자 분포 + 100만원 미만 소액 계좌 비중 기반 **보수 추정**. 시장 상승기에는 상단 가능.

### 한국 사용자 페인 — 4축

1. **거래소 분산** — 5대 거래소 투자자 1,559만 vs 거래가능 이용자 1,077만 → **평균 1.45개 거래소** 사용. Upbit 78.2% + Bithumb 19.3% 양강이지만 보조 거래소가 흔하고 글로벌 거래소(Binance, Bybit) 사용자도 다수.
2. **데이터 포맷 분열** — CSV·PDF·XLS·API 응답 형식이 거래소별 모두 다름. Upbit PDF는 텍스트 추출 필요, Binance CSV는 영문 컬럼·UTC 시각.
3. **세법 의무** — FIFO / 총평균법 선택, **의제취득가액 2026-12-31 시점 자동 적용**, 해외 거래소 환율 변환(`daily_rates` 일별), 의제 vs 실가 mix, SWAP 분리 등 한국 세법 특수 처리.
4. **도구 부재** — 한국 세법 준수 PDF 리포트를 자동 생성하는 일반 사용자용 도구가 사실상 부재 (Section 5 참조). 글로벌 도구는 한국 세법 미지원.

## 2. 솔루션

### 4단계 자동 파이프라인 (업로드 → PDF, 약 8분)

```
[1] 업로드            [2] 자동 정규화         [3] 세금 계산         [4] PDF 리포트
거래소 파일 →         FIFO/총평균 선택 →     250만 공제, 22% →    한국 세법 양식
(CSV·PDF·XLS)        의제취득가액 자동      audit trail 생성     자동 생성
                     환율 일별 자동
```

### 단계별 자동화

**1단계 — 거래소 파일 업로드**
- MVP: **Upbit (PDF)**, **Bithumb (XLS, Phase 8 1순위)**, **Binance (CSV)**
- Phase 8 확장: Coinone · Bybit · Coinbase · Gate.io · OKX · Bitget
- 파일 시그니처 검증 (무관 PDF·영수증 거절), 중복 거래 자동 dedupe

**2단계 — 자동 정규화**

| 기능 | 구현 |
|------|------|
| 거래소별 포맷 통합 | `lib/parsers/{exchange}.parser.ts` → `UnifiedTransaction` 단일 타입 |
| KST 시간대 정합 | `lib/engine/exchange-rate.ts` `kstYearOf` / `kstMonthOf` / `kstDayOf` 헬퍼 |
| 환율 자동 변환 | Supabase `daily_rates` 일별 적재 (Upbit KRW 마켓 종가, Edge function cron) |
| 의제취득가액 자동 | Supabase `deemed_cost_snapshots` + `promote_deemed_cost_from_daily_rates` 함수 (2026-12-31 자동 승격) |
| SWAP 분리 | 코인↔코인 교환을 매도+매수 두 거래로 분리 |
| Orphan 매도 처리 | 매수 기록 없는 매도 detect + 사용자 경고 (오인식 차단) |

**3단계 — 세금 계산 엔진**

| 항목 | 처리 |
|------|------|
| FIFO 엔진 | `lib/engine/fifo.ts` — lot 단위 매도 매칭, audit trail 생성 |
| 총평균 엔진 | `lib/engine/TotalAverageEngine.ts` — 연 평균 단가 (시행령 §88①) |
| 의제취득가액 | 2027.01.01 이전 매수분에 자동 적용, lot 별 `isDeemedCost` 배지 |
| 250만원 공제 | 양도차익 ₩2,500,000 초과분 자동 산출 |
| 22% 세율 | 산출세액 자동 계산 (지방세 포함) |
| 거래소별 분리 | `ExchangeCoinSummary` — 거래소 × 코인 매트릭스 |
| 이월 보유 | `holdingsAfter` — 다음 시즌 의제 플래그 유지 |

**4단계 — 한국 세법 준수 PDF 리포트**

| 페이지 | 내용 |
|--------|------|
| Cover | 신고 연도·납세자 정보·산출세액 요약 |
| Summary | 양도차익·공제·과세표준·세액 4줄 breakdown |
| Method Breakdown | FIFO 또는 총평균 흐름 (사용자가 선택) |
| Consumed Lots | 매도-매수 매칭 audit (매수일·거래소·의제 배지) |
| Deemed Cost | 의제취득가액 적용 내역 (real / estimate / user_override / missing) |
| Holdings After | 이월 보유 자산 (다음 시즌 의제 플래그) |
| Disclaimer | 사용자 자기 신고 책임 안내 |

### 신뢰 장치 — audit trail

사용자가 PDF 결과를 신뢰할 수 있도록 **모든 계산 단계의 근거 노출**:
- FIFO 매도-매수 매칭 (매수일·거래소·의제 여부)
- 의제취득가액 출처 (실가 / 추정 / 사용자 override / 누락)
- 환율 출처·갱신일·fallback 사용 여부
- 거래소별 손익 분리
- 이월 보유 자산 + 의제 플래그

### 한국 세법 전용 특수 처리

| 항목 | 구현 위치 |
|------|----------|
| 250만원 기본공제 | `lib/engine/constants.ts` `BASIC_DEDUCTION_KRW` |
| 22% 분리과세 세율 | `lib/engine/constants.ts` `CRYPTO_TAX_RATE` |
| 의제취득가액 2026-12-31 시점 | `lib/engine/exchange-rate.ts` `isPreDeemedDate` + Supabase 자동 승격 |
| KST 신고연도 경계 | `kstYearOf` 단일 헬퍼 (UTC 환경에서 1월 1일 거래 누락 방지) |
| FIFO/총평균 사용자 선택 | Settings 페이지 토글, 재계산 server action |

**글로벌 도구와의 차이**: Koinly·CoinTracker·TurboTax 등은 미국·EU 세법 기준. 한국 세법의 250만원 공제·22% 분리과세·의제취득가액 2026-12-31·KST 기준일을 동시에 지원하는 도구는 kontaxt 가 유일.

## 4. 가치 제안 — 가치 앵커 3층 구조

### 왜 3층 구조인가

대다수 SaaS는 단일 가치 앵커(예: 세무사 비용)에 의존한다. 그러나 한국 가상자산 보유자 다수는 **세무사를 직접 쓰지 않는다**. 단일 앵커만 노출하면 WTP가 ₩49,900에 도달하지 않는다.

세 앵커를 **첫 화면에서 동시 노출**해야 WTP가 ₩49,900에 도달한다. loss aversion(앵커 2)이 가장 강력하게 작동하며, 다른 두 앵커가 보강 역할을 한다.

### 앵커 1 — 세무사 비용 (gain anchor)

| 항목 | 값 |
|------|---|
| 비교 기준 | 한국 세무사 가상자산 신고 대리 비용 — **공식 평균 통계 부재** |
| 시장 정황 추정 | **₩300,000 ~ ₩1,500,000 / 건** (거래 건수·지갑 수·해외 거래소·NFT·디파이 포함 여부에 따라 견적 편차 큼) |
| Kontaxt 가격 | ₩49,900 (원타임) |
| 인식 비율 | 일부 유저 — 세무사 의뢰 경험이 있어야 인식 |

> ⚠ 한국 세무사의 **표준 가격표 자체가 없는 시장**이 kontaxt 의 차별점. 자동 도구로 표준화·예측 가능한 가격(₩49,900) 제공.

카피 예: **"세무사 위탁 견적 ₩30만~150만원의 1/10~1/30 비용으로 자동 신고서 생성."**

### 앵커 2 — 가산세·추징 회피 (loss anchor) ★ 가장 강력

가산세는 **국세기본법 §47의2~4** + 국세청 양도소득세 가산세 안내에 따라 모든 양도소득세에 동일 적용 (2027 가상자산 양도소득세도 동일 — 가상자산 전용 특수 가산세는 없음).

| 가산세 종류 | 비율 | 근거 |
|------------|------|------|
| 무신고 (일반) | 산출세액의 **20%** | 국세기본법 §47의2 |
| 무신고 (부정) | **40%** | 동 조 |
| 과소신고 (일반) | **10%** | 국세기본법 §47의3 |
| 과소신고 (부정) | **40%** | 동 조 |
| 납부지연 | 일 **0.022%** (연 환산 **8.03%**) | 국세기본법 §47의4 |

**실제 계산 예시** (양도차익 ₩10,000,000 가정):
- 산출세액 = (10,000,000 − 2,500,000) × 22% = **₩1,650,000**
- 무신고 시 추가 가산세 = 1,650,000 × 20% = **₩330,000**
- 부정 무신고 시 = 1,650,000 × 40% = **₩660,000**
- 1년 납부 지연 추가 = 1,650,000 × 8.03% ≈ **₩132,500**

→ 무신고 + 1년 지연 시 **₩462,500 추가 부담** (kontaxt 가격 ₩49,900의 **9.3배**). 자동 계산으로 이 부담을 회피.

사용자 본인 거래 데이터 기반 **회피 가능한 가산세 동적 표시** (앵커 2 핵심 UX):
- "이 신고로 가산세 ₩XXX 회피"
- "산출세액 ₩XXX × 가산세율 = 위험 ₩XXX"
- "현재 페이스 미신고 → 1년 후 ₩XXX 추가 부담"

loss aversion 톤은 사용자가 즉시 행동하게 만드는 가장 강한 동인. **결제 직전 화면에서 이 숫자를 가장 큰 비주얼**로 노출한다.

### 앵커 3 — 시간·정신 비용 (universal anchor)

| 항목 | 값 |
|------|---|
| 비교 기준 | 거래소별 데이터 정리 → FIFO 매칭 → 의제취득가액 적용 → 환율 변환 |
| 추정 소요 시간 | **8 ~ 20시간** (가상자산 자체 신고 공식 통계 부재 — 한국조세재정연구원 일반 세무 신고 평균 11.4시간 + 가상자산 다중 거래소·FIFO 매칭 복잡도 가산) |
| Kontaxt 처리 시간 | **약 8분** (파일 업로드 ~ PDF 다운로드) |
| 인식 비율 | 보편적 — 한 번이라도 수동 작업 시도한 유저는 즉시 공감 |

카피 예: **"엑셀로 8시간 vs 업로드로 8분."**

> ⚠ 자료 보강: 한양대 멘토링 시점에 베타 사용자 5~10명 자체 신고 시간 측정 권장 (실측 데이터 교체).

### 첫 화면 노출 패턴

```
[Hero]
   ↓ 큰 헤드라인 (앵커 2 — loss)
   "신고 누락 시 가산세 ₩XXX 부담. 정확히 계산해서 회피."

## 5. 경쟁 환경과 해자

### 한국 시장 — 사실상 공백 (kontaxt 선점 기회)

2025~2026 한국 가상자산 세무 SaaS 시장은 **표준 도구·표준 가격이 부재한 공백 상태**:

| 분류 | 사례 | 상태 |
|------|------|------|
| 한국 자체 가상자산 세무 SaaS | **크립토택스** (2022 해시드 시드 투자) | 시드 단계, 본격 시장 진출 미확인 |
| 한국 세무 신고 플랫폼 + 가상자산 | **삼쩜삼** | 양도세·부가세 + 세무사 연결. **가상자산 전용 기능 X**, 상담 무료 + 대행수수료 |
| 글로벌 가상자산 세무 도구 | Koinly · CoinTracker · CoinLedger · CryptoTaxCalculator · TokenTax · TurboTax | 한국 사용자 이용은 가능하나 **한국 세법 미지원** (FIFO·총평균·의제취득가액·250만 공제·22% 분리과세) |
| 한국 세무사 직접 의뢰 | — | **표준 가격표 미정립**, 견적 편차 ₩30만 ~ ₩150만+ |

→ kontaxt 는 **한국 세법 전용 + 한국 거래소 통합 + 자동 PDF 리포트** 단일 도구로서 시장 공백 채움. 2027.01.01 시행 + 2028.05 첫 신고 시즌 직전이 진입 골든 타임.

### 차별점 vs 해자

- **차별점(differentiation)** — 유저가 처음 선택하는 이유. 한국 세법 자동 계산 + 거래소 통합 + 한국 카피·UX. **카피 가능**.
- **해자(moat)** — 유저가 떠나지 못하는 이유. 기능이 아니라 시간·데이터 축적 기반. **카피 불가**.

### 진짜 해자 3가지

1. **데이터 lock-in** — 수년치 거래내역 + 취득가액 추적 누적 → 전환비용 ↑. 의제취득가액(2026-12-31 종가) 자동 승격으로 시간이 가치를 만듦. 2027~2028 첫 시즌 lock-in 사용자는 2029~2030 신고 시즌에도 자연스럽게 잔류.
2. **엣지케이스 정확도** — 거래소 10+ × 포맷 변경 × 코인 교환 × 에어드롭·하드포크·SWAP 분리 등 예외 케이스가 실제 사용자 데이터로 축적되면 후발주자가 따라잡기 어려움. MVP 시점에 Upbit/Bithumb/Binance 3종 + Phase 8 7종 = 10거래소.
3. **세무사 네트워크** — 세무사가 "이 플랫폼 리포트로 가져오세요"라고 말하는 순간 양면 네트워크 효과 발생. 2026.09~10 외부 audit 단계에서 한국 가상자산 전문 세무사 1~2명 자문 확보 (계획).

### Tax-Loss Harvesting 시뮬레이터의 위치

초기 hook 으로는 유효 (Section 9 시뮬레이터 페이지). 다만 **해자 아님** — CoinGecko API + 미실현손익 계산은 알고리즘적으로 단순. 경쟁자가 2~3주면 카피 가능.

### 포지션

**"가장 많은 한국 거래소를 가장 정확하게 처리하는 한국 세법 전용 플랫폼"** — 기능 하나가 아니라 시간 + 사용자 데이터 축적 기반. **먼저 시작해서 먼저 쌓는 것이 전부.**

## 6. 가격 책정 프레임워크

### Value-based pricing — 채택 이유

가격 책정 접근법은 세 가지:

| 접근 | 정의 | kontaxt 적용 여부 |
|------|------|-------------------|
| Cost-based | 원가 + 마진 | **❌** — SaaS는 한계비용 ~0, 원가 기반으론 가치 누락 |
| Competitor-based | 경쟁자 가격 추종 | **❌** — 한국 가상자산 세무 SaaS 시장 자체가 공백 (Section 5) |
| **Value-based** | 사용자가 얻는 가치 기준 | **✅** — 세 가치 앵커 (Section 4) 기반 |

→ kontaxt 의 ₩49,900 원타임 가격은 세무사 위탁의 1/10~1/30 + 가산세 회피 ₩462,500+ + 시간 8시간 회피의 **종합 가치**에서 도출.

### 카테고리 재정의 — 세무 서비스 (≠ 일반 구독)

**왜 카테고리가 중요한가**: 유저가 "유튜브 프리미엄 ₩14,900/월 vs kontaxt ₩89,000/년"으로 비교하면 즉시 가격 거부감. 반면 "세무사 ₩30~150만 vs kontaxt ₩49,900"으로 비교하면 1/10 가격 합리화.

| 비교 카테고리 | 결과 |
|--------------|------|
| **세무 서비스** (✅ 적정) | 삼쩜삼·TurboTax·세무사 위탁 — kontaxt 는 이 카테고리에 포지셔닝 |
| 일반 구독 (❌ 함정) | 유튜브 프리미엄·쿠팡와우·Netflix — 비교 순간 가격 폭락 |
| 포트폴리오 추적 (❌ 함정) | CoinGecko·CoinStats — 무료 대안 다수, "왜 돈 내?" 거부감 |

→ 모든 마케팅 카피·결제 페이지 카피·LOI 페이지에서 **세무 서비스 카테고리 신호 유지**.

### 가격 갭 1.8× — 결정 근거

원타임 ₩49,900 vs 연간 구독 ₩89,000 = **1.8× 갭** (₩40,000 추가).

| 시나리오 | 결과 |
|---------|------|
| 갭이 너무 좁음 (1.2~1.4×) | 신규 유저가 구독으로 쏠림 → 원타임 존재 의미 약화, 카니발라이제이션 |
| **1.8× (채택)** | 첫 결제 원타임 압도 → 1년 뒤 구독 전환 path 자연스러움 |
| 갭이 너무 넓음 (2.5×+) | 구독 가치(TLH·API·상시 대시보드) 인식 안 되면 영영 안 넘어옴 |

PMF 이후 A/B 테스트로 최적화. MVP 시점에는 1.8× 확정.

### WTP 검증 방법 — LOI (Letter of Intent)

**Van Westendorp PSM 의 한계**:
- 설문 응답 WTP vs 실제 결제 WTP 갭 **30~50%** (산업 통설)
- 50~100명 설문은 통계적 신뢰도 낮음 (특히 가상자산 세무처럼 시즌성 강한 카테고리)
- 2028.05 첫 신고 시즌까지 실제 결제 데이터 0

**LOI 로 대체** (Section 8):
- 사전 예약 결제 ₩39,900 → **카드 인증·결제까지 받음** = 진짜 WTP에 가까운 신호
- 100명 목표 = PSM 설문 50명보다 강한 데이터
- 부가 효과: 베타 사용자 풀 확보 (Phase 7 피드백 루프)

### 결제 페이지 UX — 동적 가치 노출

결제 직전 화면에서 사용자 본인 거래 데이터를 기반으로 **다음을 숫자로 표시**:

| 노출 항목 | 톤 |
|-----------|---|
| 예상 절세액 (FIFO vs 총평균 선택 시 차이) | gain |
| 가산세 회피액 (신고 누락 시 부담) | **loss (우선)** |
| 자동 계산으로 절감된 작업 시간 | universal |

단순 기능 나열(❌)이 아니라 **사용자 본인 데이터로 본인 가치를 숫자로 본다**.

```
[결제 페이지 Hero]

이 신고로 가산세 ₩872,000 회피            ← 앵커 2 (가장 큰 비주얼)

[CTA] ₩49,900 결제하기 →
[Sub] 카카오페이 · 네이버페이 · 토스페이 · 카드 일괄
