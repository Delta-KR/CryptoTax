# 시나리오 #8 — 동일 timestamp 대량 거래 합산 정확성

| 항목 | 값 |
|------|---|
| **검증 포인트** | 동일 timestamp 5건 매수의 정렬 안정성 + 합산 정확성 |
| **적용 법령** | 소득세법 시행령 §88① · §64의3② |
| **난이도** | ⭐⭐ |
| **과세 연도** | 2027년 |
| **사용자 유형** | 거주자 (개인) |
| **거래소** | Upbit 단일 |
| **자산** | BTC 단일 |

> 동일 timestamp 거래는 정렬 시 순서가 비결정적일 수 있으나, 총평균법은 합산 평균이므로 매수 순서와 무관하게 결과가 동일해야 합니다. 본 시나리오는 합산 정확성을 검증합니다.

## 3. 산식 흐름

```
[STEP 1] 동일 timestamp 5건 합산 (시행령 §88①)
  total_cost   = Σ(거래금액 + 수수료) = 5,200,000 + 5,000 = 5,205,000
  total_amount = Σ(수량) = 0.10 BTC
  avg_price    = 5,205,000 / 0.10 = 52,050,000

  ※ 정렬 순서 무관 — 합산 평균이므로 결과 동일

[STEP 2] 매도 손익
  pnl = 15,000,000 − (0.10 × 52,050,000) − 15,000 = +9,780,000

[STEP 3] 세액
  taxable_base = 9,780,000 − 2,500,000 = 7,280,000
  income_tax   = 1,456,000
  local_tax    = 145,600
  total_tax    = 1,601,600
```
