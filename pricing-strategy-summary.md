# 가격 모델 및 전략 논의 요약

> 2026.05.20 — Claude Code 컨텍스트 전달용

---

## 1. 가격 책정 프레임워크

- **Value-based pricing 채택**: 원가 기반(cost-based)이나 경쟁자 기반(competitor-based)이 아닌, 유저가 얻는 가치 기준으로 가격 설정
- **가치 앵커(value anchor)**: 세무사 대리 비용(건당 20~50만원)이 비교 기준점
  - 세무사 비용의 1/4~1/3 수준이 적정 포지션
- **유튜브/쿠팡와우 같은 일반 구독 서비스와 비교하면 안 됨** — 카테고리가 다름. 비교 대상은 세무 서비스(삼쩜삼, TurboTax 등)

### 가격 검증 방법론
- **Van Westendorp PSM (Price Sensitivity Meter)**: 커뮤니티(땡글, 코인판 등)에서 50~100명 설문으로 적정 가격대 도출 가능
- MVP 전에 실행 권장

## 2. 과금 구조 결정

### 기존 계획의 문제점
- 월 구독(₩4,900/월)은 연 1회 사용 서비스와 사용 패턴 불일치 → 11개월간 "왜 돈 나가지?" 거부감
- 프리미엄 구독(₩19,900/년)과 원타임(₩29,900~49,900)이 공존하면 카니발라이제이션 발생
- ₩19,900/년은 세금 도구치고 너무 저가 → 신뢰도 저하 시그널

### 확정 구조: 원타임 + 연간 구독 이중 모델

**원타임 (₩49,900~79,000)**
- 거래내역 업로드, 세금 계산, PDF 리포트, FIFO/이동평균 비교
- 매년 신고 시즌(5월) 한 번 결제
- 매출의 base, MVP에서 먼저 검증

**연간 구독 (₩59,000~99,000/년)**
- 원타임 기능 전부 포함
- 추가 제공: 상시 세금 포지션 대시보드(실현 손익 실시간 추적), Tax-Loss Harvesting 알림, 의제취득가액 최적화 시뮬레이션, 거래소 API 자동 연동
- 핵심 명분: "포트폴리오 추적"이 아닌 **"연중 절세 도구"** — 기존 무료 포트폴리오 앱(CoinGecko, CoinStats 등)과 경쟁 회피

### 월 결제 옵션 관련
- 월 결제를 두면 시즌 체리피킹 리스크 (5월만 결제하고 해지)
- 제공한다면 극단적으로 높게 설정 (예: 월 ₩19,900 vs 연 ₩99,000)해서 사실상 연간으로 유도
- 또는 연간 결제만 제공하는 것도 옵션
- 참고: 연간 구독도 MRR = Annual ÷ 12로 계산하므로 MRR 안정성에 문제 없음

## 2-1. 티어별 상세 비교 및 미해결 과제

### 원타임 (₩49,900~79,000) — 신고 시즌 1회 결제
| 기능 | 포함 여부 |
|------|-----------|
| 거래내역 파일 업로드 (PDF/XLS/CSV) | ✅ |
| 세금 자동 계산 (FIFO/이동평균 선택) | ✅ |
| 의제취득가액 자동 적용 | ✅ |
| 해외 거래소 환율 변환 | ✅ |
| PDF 세금 리포트 생성 | ✅ |
| 거래소 API 자동 연동 | ❌ |
| 상시 실현 손익 대시보드 | ❌ |
| Tax-Loss Harvesting 알림 | ❌ |

- **유저 매리트**: 세무사 비용(20~50만원)의 1/4~1/3 가격, 필요할 때만 결제, 구독 피로감 제로
- **타겟 유저**: 연 1회 신고만 하면 되는 일반 투자자

### 연간 구독 (₩59,000~99,000/년) — 원타임 전체 포함 + 상시 기능
| 기능 | 포함 여부 |
|------|-----------|
| 원타임 기능 전부 | ✅ |
| 거래소 API 자동 연동 (파일 업로드 불필요) | ✅ |
| 실현 손익 실시간 추적 대시보드 | ✅ |
| Tax-Loss Harvesting 알림 ("이 코인 지금 팔면 세금 ₩XX 절감") | ✅ |
| 의제취득가액 최적화 시뮬레이션 | ✅ |

- **유저 매리트**: 연중 절세 기회 포착, 파일 업로드 없이 자동 동기화, 매도 타이밍에 세금 영향 미리 확인
- **타겟 유저**: 거래 빈번한 트레이더 또는 보유 자산이 큰 홀더

### ⚠️ 미해결 과제 — 구현 전 검증 필요

**1. 가격 갭 설계**
- 원타임 ₩79,000 vs 구독 ₩99,000 → 차이 ₩20,000뿐이면 대부분 구독 선택 (원타임 존재 의미 약해짐)
- 원타임 ₩49,900 vs 구독 ₩99,000 → 차이 ₩50,000이면 구독 전환율 저하
- **결론**: 가격 갭은 MVP 원타임 출시 후 A/B 테스트로 검증

**2. 구독 타겟 유저 분리 가능성**
- API 자동 연동을 원하는 유저 = 거래 빈번한 트레이더
- Tax-Loss Harvesting이 의미 있는 유저 = 보유 자산이 큰 장기 홀더
- 이 두 그룹이 같은 사람이 아닐 수 있음 → 구독 티어 내에서 가치 제안이 분산될 위험
- **결론**: 구독 출시 시 어느 쪽이 핵심 가치인지 유저 인터뷰로 확인 필요

**3. 단순 포트폴리오 추적은 차별점 아님**
- 업비트/빗썸 앱 자체 포트폴리오, CoinGecko, CoinStats 등 무료 대안 다수
- 구독의 핵심 명분은 "포트폴리오 보기"가 아닌 **"연중 절세 도구"**여야 함

1. **MVP는 원타임 결제만** — 이중 구조를 동시에 만들지 않음

## 4. 경쟁 전략 및 해자(Moat)

### 차별점 vs 해자 구분
- **차별점(differentiation)** = 유저가 처음 선택하는 이유 (초기 유입용)
- **해자(moat)** = 유저가 떠나지 못하는 이유 (방어 가능한 포지션)

### Tax-Loss Harvesting 시뮬레이터의 위치
- 초기 유입용 후크(hook)로는 유효
- 해자는 아님 — 경쟁자가 2~3주면 카피 가능 (CoinGecko API + 미실현손익 계산은 알고리즘적으로 단순)
- 차별점이라고 확신하면 안 됨

### 진짜 해자 3가지
1. **데이터 lock-in**: 유저가 수년치 거래내역을 업로드하고 취득가액이 추적되면 타 서비스 전환비용이 높아짐
2. **엣지케이스 정확도**: 거래소 10개 × 파일 포맷 변경 × 코인간 교환 × 에어드롭 × 하드포크 등 예외 케이스가 실제 유저 데이터로 축적되면 후발주자가 따라잡기 어려움
3. **세무사 네트워크**: 세무사가 "이 플랫폼 리포트로 가져오세요"라고 말하는 순간 양면 네트워크 효과 발생 — 기능으로 카피 불가

### 핵심 결론
- 방어 가능한 포지션 = **"가장 많은 거래소를 가장 정확하게 처리하는 플랫폼"**
- 기능 하나가 아니라 시간 + 유저 데이터 축적으로 만들어짐
- **먼저 시작해서 먼저 쌓는 것이 전부**

## 6. MVP 직전 확정사항 (2026-05-21 결정)

[[pricing-strategy-review-2026-05-20]] 비판적 검토에서 도출된 6건 미정의 항목 + 가격 수치를 사용자 결정으로 확정. 모든 사업계획서 후속 수정의 기준점.

### 6.1 가격 수치 (MVP)
- **원타임 ₩49,900** — 진입장벽 낮춰 첫 결제 압도
- **연간 구독 ₩89,000/년** — 원타임의 1.8× 갭 → 업그레이드 ₩40K 추가 느낌
- 결정 근거: 5만원 미만 진입 저항 낮음 + 9만원대 구독은 카니발라이제이션 회피
- 추후 PMF 후 A/B 테스트로 최적화

### 6.2 가치 앵커 3층 구조 → Section 04 반영
세무사 비용 단일 의존에서 3층 구조로:
1. **세무사 비용** (건당 20~50만원) — 일부 유저만 인식
2. **가산세·추징 회피** (loss aversion) — 모두에게 작동, 가장 강력
3. **시간·정신 비용** — 약하지만 보편적

첫 화면에서 세 앵커 동시 노출. 단일 앵커만 쓰면 WTP가 ₩49,900에 못 미침.

### 6.3 결제 페이지 가치 노출 UX → Section 07 반영
- 사용자 거래 데이터 기반 **예상 절세액 · 가산세 회피액 동적 표시**
- loss aversion 톤: "정확히 안 내면 가산세 + 추징"
- gain 톤: "이 도구로 합법적으로 줄여줌"
- 결제 직전 화면에서 숫자로 가치를 보여줌 (단순 기능 나열 ✗)

### 6.4 무료↔유료 paywall = PDF 다운로드 지점 → Section 07 반영

| 단계 | 무료/유료 |
|------|---------|
| 거래내역 업로드 + 파싱 | 무료 |
| 세금 추정치 표시 | 무료 |
| 계산 흐름 · 거래 내역 확인 | 무료 |
| **PDF 리포트 다운로드** | **유료 wall** |
| 코인별 손익 상세 | 유료 |
| 거래소별 분리·매칭 audit | 유료 |
| FIFO/이동평균 비교 선택 | 유료 |

매몰비용 효과 — 유저가 다 해놓고 마지막 결과만 결제. 전환율 극대화 패턴.

### 6.5 결제 인프라 = 포트원 → Section 07 반영
- **선택**: 포트원 (통합 게이트웨이)
- 한 번 연동으로 카카오페이·네이버페이·토스페이·신용카드 일괄 커버
- 한국 인디/스타트업 표준 — 토스페이먼츠 대비 구현 난이도 낮음
- 수수료는 표준 PG + 포트원 마진 (가입 후 구체 수치 확인)
- v2 후보로 토스페이먼츠 직접 연동 (안정성·공식성 필요해질 때)

### 6.6 사전 예약 결제 (LOI) → Section 08 반영
- **운영 기간**: 2026.07 ~ 2026.12 (Phase 7~2 기간)
- **가격**: ₩39,900 (정가 ₩49,900 대비 20% 얼리 인센티브)
- **방식**: 카드 인증·결제 → 첫 신고 시즌(2027.05) 사용 → 환불 자유
- **목표**: 100명
- 효과: PSM 설문 대비 진짜 WTP 검증 + 베타 유저 풀 확보
- 30일 환불 보증으로 결제 저항 완화

### 6.7 연중 유입 hook = 양도 시뮬레이터 무료 분리 → Section 09 반영
